# Contributing to Utvekslingsportalen

Thank you for your interest in contributing to **Utvekslingsportalen**!  
Contributions, feedback, and suggestions are very welcome.

## How to contribute

### Reporting issues
If you find a bug, incorrect data, or have a feature suggestion:
- Check existing issues to avoid duplicates
- Open a new issue and provide as much relevant detail as possible

### Proposing changes
For code or content changes:
1. Fork the repository
2. Create a new branch from `main`
3. Make your changes with clear, descriptive commit messages
4. Open a Pull Request describing what you changed and why

### Data contributions
If you are contributing exchange data or corrections:
- Ensure the data is accurate and clearly sourced
- Follow the existing data structure and naming conventions
- Avoid including personal or sensitive information

## Code style and quality
- Keep changes focused and minimal
- Follow the existing coding style and project structure
- Test changes locally where applicable

## Code of Conduct
This project follows the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md).  
By participating, you agree to uphold this standard.

## Questions
If you are unsure about anything, feel free to open an issue for discussion before starting work.
